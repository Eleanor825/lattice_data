Run It Locally

These are the exact commands behind the current demo outputs. 

PYTHONPATH=src python3 -m lattice demo \
  --raw-output data/demo_raw/solid_state \
  --compiled-output data/demo_compiled/solid_state \
  --domain materials \
  --dataset-name Lattice-Materials-RealDemo \
  --query "solid state battery electrolyte" \
  --compound "lithium iron phosphate" \
  --compound "lithium cobalt oxide"
PYTHONPATH=src python3 -m lattice engine-compile \
  --engine local \
  --input examples/runtime/raw \
  --output outputs/runtime-local \
  --domain materials \
  --dataset-name Lattice-Runtime-Local

PYTHONPATH=src python3 -m lattice engine-compile \
  --engine spark \
  --input examples/runtime/raw \
  --output outputs/runtime-spark \
  --domain materials \
  --dataset-name Lattice-Runtime-Spark

Engine Status

Lattice already has executable engine support. Local and Spark are verified in this repository. Flink has a compatible code path, but the current local runtime is still blocked.

Engine Available Verified Current State
Local Ready Yes Default path for local development and debugging.
Spark Ready Yes Local Spark mode is verified and produces the same runtime result shape.
Flink Partial No Code path exists, but the current local PyFlink runtime is still blocked by environment dependencies.
The platform story is still valid: Lattice already runs locally and through Spark. Flink compatibility is implemented at the code path level, but its local runtime still needs a cleaner environment or containerized setup to fully validate.
