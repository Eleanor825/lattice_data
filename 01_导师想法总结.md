# Lattice 项目：导师想法总结

## 一句话结论

你导师真正想做的，不是“再做一个自动化 agent”，而是做一个**以数据为核心资产**的系统：能够从异构来源自动提取、清洗、结构化、评估并组合高价值训练数据，最终服务于 foundation model 的 pretraining、continued pretraining、fine-tuning 和 post-training。

## 导师最看重的 6 件事

### 1. 核心目标是“数据”，不是“agent”

- Agent 只是自动化界面，不是长期壁垒。
- 真正会沉淀价值的是：
  - 数据本身
  - 操作数据的 library / toolkit
  - 关于“什么数据更有价值”的 formalization
- 原因很直接：模型和 agent 能力会持续被更强通用 AI 吞并，但高质量数据资产、数据 schema、数据操作系统、数据价值度量会长期留下来。

### 2. 要做的是端到端的数据基础设施

导师想要的不是“收集一些数据然后清理一下”，而是一条完整链条：

1. 从网页、论文、知识库、垂类站点等来源抓取信息
2. 把异构原始内容转成可训练、可比较、可组合的数据对象
3. 用统一工具链对这些数据做过滤、重写、去重、标注、结构化、合成
4. 最后自动决定哪些数据该进训练、占多大比例、以什么顺序喂给模型

也就是说，他想做的是一个 **data operating system**，而不是一个单点脚本。

### 3. 要沉淀“最小可覆盖”的 data schema

导师明确提到，不同数据形态不可能被一个格式统一，但应该有一个**minimal schema set**，覆盖绝大多数训练数据：

- text corpus
- classification / labeling data
- instruction / preference data
- structured or semi-structured data
- 领域知识抽取后的 record / tuple / graph

理想状态是：

- 新数据进入系统后先自动匹配某个 schema
- 再调用对应处理 pipeline
- 最终进入统一的数据资产库

这件事本质上是在做“数据层的接口标准化”。

### 4. 研究核心是“数据价值/知识密度”的 formalization

导师反复强调的一点是：

- 不同来源的数据价值不一样
- 这种差异现在大多靠人拍脑袋
- 缺少一个严格、可计算、可优化的 formalization

他想问的是：

- 从 Expedia 抓来的信息和从 paper 抓来的信息，谁更值钱？
- 什么叫知识密度更高？
- 什么样的数据更能提升某个模型、某个任务、某个阶段的训练？

这意味着项目的研究主轴不是“怎么抓”，而是“怎么量化数据的质量、信息量、任务相关性和训练价值”。

### 5. 最终要走向自动的数据合成与 feeding strategy

导师提出的更深层目标是：

- 数据选择不是“全扔进去训一遍”
- 而是要自动决定：
  - 选择哪几类数据
  - 各自占多大比例
  - 是否需要合成新数据
  - 训练过程中按什么顺序喂给模型
  - 不同阶段是否切换 mixture / curriculum / batch composition

他把这个问题看作一个**组合优化问题**：

- 在固定资源下
- 给定任务目标和数据特征
- 自动搜索最优数据集、最优配比、最优训练顺序

### 6. 希望形成可见的数据资产和社区入口

导师还提了一个很现实的方向：

- 不要总是每次做项目都从头到处搜数据
- 应该把某个方向的数据先“扣拢下来”
- 做成别人会来访问、复用、持续关注的资产库 / 网站 / Hub

这说明他想做的不只是论文，也包括：

- 可持续积累的数据仓库
- 垂直方向的数据门户
- 对外可展示、可传播、可吸引合作的数据平台

## 导师不想做什么

### 1. 不想只做“数据清洗工程”

如果只是抓数据、洗数据、存起来，这在他看来想得太浅。

### 2. 不想把“agent”当成唯一卖点

因为 agent 能力很快会被更强模型商品化，单纯 automation layer 不够稳固。

### 3. 不想停留在 heuristic

目前很多数据配比、课程学习顺序、数据合成比例都靠经验规则。导师想要的是：

- 有理论或准理论的量化指标
- 有系统性的 decision rule
- 有自动化优化，而不是人工拍参数

## 这件事可以拆成 3 个层次

### A. 资产层

- 构建高价值、可持续更新、可复用的领域数据仓库
- 例如材料、科学、技术文档、工具使用轨迹、结构化知识等

### B. 系统层

- 做一个处理异构数据的 library / platform
- 包含 schema、解析、去重、过滤、重写、合成、打包、追踪

### C. 研究层

- 定义数据价值
- 建立数据选择与 mixing strategy 的 formalization
- 把数据合成与训练 feeding 变成可优化问题

## 如果把导师的话翻译成一个正式研究问题

可以表述为：

> 能否构建一个面向 foundation model 的数据操作系统，将异构原始信息自动映射为统一数据对象，并通过任务条件下的数据价值建模，自动完成数据筛选、配比、合成和训练喂养策略优化，从而在固定资源预算下最大化模型能力提升？

## 最适合立项的主线

最适合做成 research proposal 的，不是“做一个 agent”，而是下面这条主线：

> **Lattice = 统一数据 schema + 数据价值建模 + 自动数据组合/训练策略优化**

其中：

- `schema/library` 是工程抓手
- `data valuation` 是研究核心
- `agent` 是可选自动化层

## 我对导师诉求的收敛版本

如果只用一句更学术的话概括：

> 这是一个 **data-centric foundation model research** 项目，目标是把“高价值训练数据的构造与使用”从经验驱动，推进到可表示、可度量、可优化、可自动化的系统范式。
